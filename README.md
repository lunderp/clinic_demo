# 始末诊所

《始末诊所》是一款使用 **Godot** 开发的剧情驱动型心理诊疗题材游戏。玩家将扮演心理医生陈默，在接待患者、问诊选择、资源管理与剧情推进中，逐步接近诊所、AI 伴侣“莫矢”以及主角自身状态背后的真相。

## 项目状态

> 当前项目处于原型开发阶段。

* 引擎：Godot 4.6.3
* 类型：剧情向 SLG / AVG
* 平台：PC
* 当前目标：完成一个 15～30 分钟的可游玩 Demo
* 当前进度：主菜单、场景切换、黑屏过渡、基础患者数据结构开发中

## 一句话介绍

玩家将扮演心理医生，通过问诊、判断与治疗患者，在资源压力与剧情谜团中推进故事。

## 核心玩法循环

```text
接待患者 → 问诊选择 → 诊断/处理 → 资源变化 → 病历记录 → 进入下一天
```

玩家需要在每日诊疗中处理不同患者的问题，同时管理自身压力、金钱与剧情进展。患者并非单纯的任务对象，而是推动世界观、主角状态与核心谜团展开的重要角色。

## 设计方向

《始末诊所》以心理诊疗为核心外壳，结合剧情推进、资源管理与角色分支，目标是让玩家在“帮助他人”与“承受压力”之间持续做出选择。

游戏前期更接近医生模拟器：玩家接待患者、进行问诊、积累金钱，并尝试达成一个看似明确的目标。随着剧情推进，玩家会发现部分目标无法被正常达成，诊所、莫矢与主角陈默之间的关系也逐渐出现异常。

中后期游戏会逐渐转向更高压的剧情体验。社会环境变化、患者数量增加、主角状态恶化，玩家需要在更紧张的节奏中继续完成诊疗，并逐步接触到主角过往与 AI 相关的核心真相。

## 主要系统

### 患者问诊系统

每位患者拥有独立的对话节点、选项分支与诊疗结果。玩家的选择会影响患者记录、资源变化与后续剧情。

### 资源系统

当前计划包含：

* 金钱
* 压力
* 心情 / 状态
* 患者记录

这些资源不会只作为数值存在，而是用于推动玩家在剧情和经营之间做取舍。

### 病历记录系统

玩家接待过的患者会留下记录。病历既是收集内容，也是回顾剧情、理解患者与世界观的重要入口。

### 场景切换与演出

当前已实现全局场景管理器雏形，通过 `SceneManager`、`ColorRect` 与 `AnimationPlayer` 实现场景切换时的黑屏过渡。

## 美术方向

本作整体美术倾向为：

* 粗外轮廓
* 少内线
* 低饱和色彩
* 较低细节密度
* 偏压抑但不过度沉重的视觉氛围

心理题材需要一定严肃感，但游戏体验不能长期处于单一压抑状态。因此角色互动、部分患者表现与莫矢的性格设计会承担调节节奏的作用。

## AI 辅助说明

本项目在开发过程中使用 AI 辅助进行资料整理、问题排查、设计讨论、代码学习与开发规划。项目核心方向、设计判断与最终取舍由作者完成。

AI 在本项目中的定位是辅助工具，而不是替代创作与开发决策。
